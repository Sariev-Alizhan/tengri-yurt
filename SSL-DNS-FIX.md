# Исправление ошибки SSL (SERVFAIL, DNS) для tengri-yurt.kz

Ошибка **«SERVFAIL looking up A for tengri-yurt.kz»** значит: в интернете домен пока не указывает на твой хостинг (или зона DNS не создана/настроена неправильно). Пока это не исправить, Let's Encrypt не выдаст сертификат.

---

## Шаг 1: Где куплен домен tengri-yurt.kz?

- **Если домен куплен в PS Cloud (ps.kz)** — настраиваем всё в личном кабинете ps.kz (шаги ниже).
- **Если домен куплен у другого регистратора** — там нужно указать **NS-серверы PS Cloud** и при необходимости A-запись (см. раздел «Домен у другого регистратора» в конце).

---

## Шаг 2: Проверить NS-серверы (если домен в PS Cloud)

1. Зайди на **https://www.ps.kz** → личный кабинет.
2. Раздел **«Домены»** → выбери **tengri-yurt.kz**.
3. Убедись, что для домена указаны NS:
   - **ns1.ps.kz**
   - **ns2.ps.kz**  
   Если стоят другие — смени на эти и сохрани.

---

## Шаг 3: Создать или проверить DNS-зону (если домен в PS Cloud)

Без зоны DNS домен никуда не «смотрит», поэтому и возникает SERVFAIL.

1. В карточке домена **tengri-yurt.kz** найди пункт **«DNS-зона»** / **«Управление зоной»** / **«Создать зону»**.
2. Если зоны ещё нет — нажми **«Создать зону»**. Укажи:
   - домен: **tengri-yurt.kz**
   - IP сервера: **тот, что даёт хостинг PS Cloud** (смотри в Plesk или в разделе «Хостинг» в личном кабинете — там обычно указан IP твоего сервера).
3. В зоне должны быть записи:
   - **A** для **@** (или пустое имя) → IP твоего хостинга.
   - По желанию **A** или **CNAME** для **www** → тот же хостинг (или тот же IP).

Точные названия кнопок могут отличаться (например: «Добавить запись», «A-запись»). Главное — чтобы для **tengri-yurt.kz** (корень) была **A-запись на IP хостинга**.

---

## Шаг 4: Узнать IP хостинга

- В **Plesk**: зайди в раздел «Сайты и домены» → домен tengri-yurt.kz → часто там показывают IP или «Хостинг» с адресом.
- Или в личном кабинете **ps.kz** → твой виртуальный хостинг → в описании/настройках должен быть **IP-адрес сервера**. Его и подставляй в A-запись.

---

## Шаг 5: Подождать обновления DNS

После сохранения зоны и NS обновление занимает от **15–30 минут** до **24 часов**.  

Проверить можно так:
- Зайди на **https://dnschecker.org**
- Введи **tengri-yurt.kz**, тип **A** — должны появиться ответы с твоим IP (зелёные галочки в разных странах).

Когда A-запись начнёт отвечать по всему миру — можно снова запрашивать сертификат в Plesk.

---

## Шаг 6: Повторно выпустить сертификат в Plesk

1. Plesk → **SSL/TLS-сертификаты** для **tengri-yurt.kz**.
2. Выбери **Let's Encrypt** → установить/выпустить сертификат.
3. Если DNS уже в порядке, выпуск пройдёт без ошибки SERVFAIL.

---

## Если домен зарегистрирован у другого регистратора

1. В панели **регистратора домена** (где покупал tengri-yurt.kz) открой настройки DNS/делегирования.
2. Укажи **NS-серверы**:
   - **ns1.ps.kz**
   - **ns2.ps.kz**
3. Либо, если регистратор не даёт менять NS, создай у него **A-запись** для **@** и при необходимости для **www** на **IP твоего хостинга PS Cloud** (его даёт Plesk/ps.kz).
4. После сохранения подожди распространения DNS (до 24 часов) и снова запроси Let's Encrypt в Plesk.

---

Кратко: ошибка «SERVFAIL» исчезнет, когда для **tengri-yurt.kz** в интернете будет корректная A-запись на IP твоего хостинга и при необходимости правильные NS (ns1.ps.kz, ns2.ps.kz). После этого SSL в Plesk выпустится без этой ошибки.
